# 后台等待事件数

本节为您介绍 OceanBase 租户的监控指标后台等待事件数，该指标细分为 11 个指标，分别为 on_cpu（后台处于 on_cpu 状态的等待事件数）、application（后台处于 application 状态的等待事件数）、configuration（后台处于 configuration 状态的等待事件数）、administrative（后台处于 administrative 状态的等待事件数）、concurrency（后台处于 concurrency 状态的等待事件数）、commit（后台处于 commit 状态的等待事件数）、network（后台处于 network 状态的等待事件数）、user_io（后台处于 user_io 状态的等待事件数）、system_io（后台处于 system_io 状态的等待事件数）、cluster（后台处于 cluster 状态的等待事件数）、other（后台处于 other 状态的等待事件数）。您可指定统计范围，查询租户在单个 Zone 或 OBServer 节点的 **后台等待事件数** 数据。

## on_cpu

### 指标介绍

统计范围内，租户后台处于 on_cpu 状态的等待事件数。

### 指标参数说明

| **指标项** |     **指标名称**      | **单位** |
|---------|-------------------|--------|
| on_cpu     | ob_background_on_cpu_waitevent_cnt |   个    |

### 计算表达式

sum(ob_background_waitevent_cnt{wait_class="ON_CPU",@LABELS}) by (@GBLABELS)
