# SQL 执行事件数

本节为您介绍 OceanBase 租户的监控指标 SQL 执行事件数，该指标细分为 13 个指标，分别为 in_parse（正在执行 SQL PARSE 的活跃会话数）、in_pl_parse（正在执行 PL PARSE 的活跃会话数）、in_plan_cache（正在访问 plan cache 的活跃会话数）、in_sql_optimize（正在执行 SQL 优化的活跃会话数）、in_sql_execution（正在执行 SQL 语句的活跃会话数）、in_px_execution（正在进行 PX SQL 并行执行的活跃会话数）、in_sequence_load（正在进行自增列或 SEQUENCE 取值的活跃会话数）、in_committing（正处于事务提交阶段的活跃会话数）、in_storage_read（正在执行存储层读的活跃会话数）、in_storage_write（正在执行存储层写的活跃会话数）、in_remote_das_execution（正在远程 DAS 执行的活跃会话数）、in_plsql_compilation（正在 PL 编译的活跃会话数）、in_plsql_execution（处于 plsql execution 状态的执行事件数）。您可指定统计范围，查询租户在单个 Zone 或 OBServer 节点的 **SQL 执行事件数** 数据。

## in_parse

### 指标介绍

统计范围内，正在执行 SQL PARSE 的活跃会话数。

### 指标参数说明

| **指标项** |     **指标名称**      | **单位** |
|---------|-------------------|--------|
| in_parse     | sql_event_in_parse |   个    |

sum(ob_sql_event_in_parse{@LABELS}) by (@GBLABELS)
