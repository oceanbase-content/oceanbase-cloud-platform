# 如何定位租户下执行时间长的慢 SQL

慢 SQL 会给业务带来不好的体验，同时也会消耗数据库大量的资源，从而造成数据库不稳定。因此，我们需要尽量从各种维度来降低慢 SQL 的数量。
假设需要定位执行时间超过 100ms 的 SQL，可以先从 **列管理** 中勾选 **最大响应时间** 列，然后在 TopSQL 或者 SlowSQL 高级条件中输入：**最大响应时间 >=  100ms**。
